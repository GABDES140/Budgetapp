# 04 - Parcours Utilisateur

## Objectif

Décrire les principaux flux utilisateurs de BudgetApp afin que Codex comprenne l'expérience attendue et les écrans nécessaires.

---

# Parcours 1 - Création de compte et première connexion

1. L'utilisateur arrive sur la page d'accueil ou de connexion.
2. Il crée un compte.
3. Il est redirigé vers l'application.
4. Le dashboard s'affiche avec un état vide propre.
5. L'utilisateur peut ajouter une première transaction, configurer un budget ou importer un fichier.

## États vides attendus

Les états vides doivent être utiles et visuels. Ils doivent expliquer l'action suivante sans générer de surcharge.

Exemple :

- Aucun revenu ajouté.
- Aucune dépense enregistrée.
- Aucun objectif configuré.

Chaque état vide doit proposer une action claire.

---

# Parcours 2 - Ajouter une dépense rapidement

1. L'utilisateur clique sur un bouton principal « Ajouter ».
2. Il choisit dépense ou revenu.
3. Il entre le montant.
4. Il choisit une catégorie.
5. Il confirme.
6. Le dashboard et les analytics se mettent à jour.

## Variante détaillée

L'utilisateur peut ouvrir les champs avancés :

- description ;
- sous-catégorie ;
- note ;
- devise ;
- date ;
- récurrence.

---

# Parcours 3 - Consulter le dashboard

1. L'utilisateur arrive sur la page Dashboard.
2. Il voit les KPI principaux.
3. Il consulte les graphiques revenus vs dépenses.
4. Il repère les catégories dominantes.
5. Il consulte les transactions récentes.
6. Il ajuste les widgets au besoin.

---

# Parcours 4 - Personnaliser le dashboard

1. L'utilisateur ouvre le mode personnalisation.
2. Il active ou désactive des widgets.
3. Il déplace les widgets.
4. Il sauvegarde la configuration.
5. Le dashboard reflète ses préférences.

---

# Parcours 5 - Analyser les finances

1. L'utilisateur ouvre la page Analytics.
2. Il choisit une période.
3. Il applique des filtres.
4. Il consulte les graphiques.
5. Il compare mois/mois ou année/mois.
6. Il identifie les tendances et postes de dépenses.

---

# Parcours 6 - Créer un objectif financier

1. L'utilisateur ouvre la page Objectifs.
2. Il clique sur « Nouvel objectif ».
3. Il définit un nom, un montant cible, une devise et une date optionnelle.
4. Il sauvegarde.
5. L'objectif apparaît dans la liste et sur le dashboard.
6. Sa progression est visible.

---

# Parcours 7 - Importer des transactions

1. L'utilisateur ouvre Import / Export.
2. Il télécharge le template.
3. Il remplit le fichier.
4. Il importe le CSV ou Excel.
5. L'application valide les colonnes.
6. L'utilisateur prévisualise les données.
7. Il confirme l'import.
8. Les transactions sont ajoutées.

---

# Parcours 8 - Exporter les données

1. L'utilisateur ouvre Transactions ou Import / Export.
2. Il applique des filtres si désiré.
3. Il clique sur Exporter.
4. Il choisit CSV ou Excel.
5. Le fichier est généré.

---

# Parcours 9 - Budget partagé

1. L'utilisateur ouvre la section Budget partagé.
2. Il entre le courriel d'une personne.
3. L'application crée une invitation.
4. La personne invitée accepte.
5. Les deux utilisateurs accèdent au même budget.

---

# Parcours 10 - Mode sombre

1. L'utilisateur ouvre Paramètres.
2. Il choisit light mode ou dark mode.
3. Le thème est appliqué immédiatement.
4. La préférence est sauvegardée.
